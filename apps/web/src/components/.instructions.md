# Component Development Guidelines

When creating new components or refactoring existing pages:

## 1. Feature Folder Structure

Create a new feature folder under `src/components/[Feature]/`:

```
components/[Feature]/
├── [FeatureName].tsx          # Smart component (if needed)
├── [Component].tsx             # Presentational components (multiple allowed)
└── index.ts                    # Export public API
```

## 2. Smart Component Pattern

**Smart components** use hooks and manage data fetching, caching, and error handling.

```typescript
// components/[Feature]/[Feature].tsx
import { use[Feature] } from '../../hooks/use[Feature]';
import { [Component] } from './[Component]';

export function [Feature]() {
  const { data, isLoading, error } = use[Feature]();
  
  // Manage loading, error, empty states
  if (isLoading) return <div role="status">{t('loading')}</div>;
  if (error) return <div role="alert">{t('error')}</div>;
  if (!data?.length) return <div>{t('empty')}</div>;
  
  // Render child components with data
  return (
    <div role="list">
      {data.map(item => <Component key={item.id} item={item} />)}
    </div>
  );
}
```

## 3. Presentational Component Pattern

**Presentational components** are pure UI - no hooks except `useTranslation` and `useNavigate`.

```typescript
// components/[Feature]/[Component].tsx
import { useTranslation } from 'react-i18next';

interface ComponentProps {
  data: DataType;
  onAction?: (id: string) => void;
}

export function Component({ data, onAction }: ComponentProps) {
  const { t } = useTranslation();
  
  return (
    <article>
      <h3>{data.name}</h3>
      <button onClick={() => onAction?.(data.id)}>
        {t('action')}
      </button>
    </article>
  );
}
```

## 4. Hook Pattern

Create a hook in `src/hooks/use[Feature].ts`:

```typescript
// hooks/use[Feature].ts
import { useQuery } from '@tanstack/react-query';
import { supabase } from '../lib/supabase';
import { useTenant } from './useTenant';
import { [Schema] } from '../schemas';
import { z } from 'zod';

export interface Use[Feature]Result {
  data: DataType[];
  isLoading: boolean;
  error: Error | null;
}

export function use[Feature](): Use[Feature]Result {
  const tenant = useTenant();
  
  const { data = [], isLoading, error } = useQuery({
    queryKey: ['[feature]', tenant?.id],
    queryFn: async () => {
      if (!tenant?.id) throw new Error('Tenant not resolved');
      
      const { data, error: err } = await supabase
        .from('[table]')
        .select('...')
        .eq('tenant_id', tenant.id);
      
      if (err) throw err;
      return z.array([Schema]).parse(data || []);
    },
    enabled: !!tenant?.id,
    staleTime: 5 * 60 * 1000,
  });
  
  return { data, isLoading, error: error instanceof Error ? error : null };
}
```

## 5. Schemas (Validation)

Define validation schemas in `src/schemas/[feature].ts` or centralize in existing schema files:

```typescript
// schemas/[feature].ts
import { z } from 'zod';

export const [FeatureSchema] = z.object({
  id: z.string().uuid(),
  name: z.string().min(1),
  // ... other fields
});

export type [Feature] = z.infer<typeof [FeatureSchema]>;
```

Then export from `schemas/index.ts`:

```typescript
export { [FeatureSchema], type [Feature] } from './[feature]';
```

## 6. Exporting from index.ts

Always export public components and hooks from `index.ts`:

```typescript
// components/[Feature]/index.ts
export { [Feature] } from './[Feature]';
export { [Component] } from './[Component]';
```

Then import in pages and other components:

```typescript
import { [Feature] } from '../components/[Feature]';
```

## 7. Accessibility Requirements

Every component must include WCAG 2.1 Level AA patterns:

**Loading State**:
```tsx
<div role="status" aria-live="polite" aria-label={t('loading')}>
  {t('common.loading')}
</div>
```

**Error State**:
```tsx
<div role="alert" aria-live="assertive">
  {error.message}
</div>
```

**Form Labels**:
```tsx
<label htmlFor="field">{t('form.field')}</label>
<input
  id="field"
  aria-invalid={!!error}
  aria-describedby={error ? 'field-error' : undefined}
/>
```

**Button Focus**:
```tsx
<button className="... focus-visible:outline-2 outline-primary outline-offset-2">
  {t('action')}
</button>
```

**List Items**:
```tsx
<div role="list">
  {items.map(item => (
    <div key={item.id} role="listitem">
      <Item item={item} />
    </div>
  ))}
</div>
```

## 8. Code Limits

Keep code size reasonable:

- **Pages**: < 50 lines (light composition only)
- **Smart Components**: < 100 lines
- **Presentational Components**: < 80 lines
- **Hooks**: < 60 lines (extract complex logic to utilities)

## 9. No Magic Rules

**Forbidden patterns**:
- ❌ `any` type (use `unknown` or proper type)
- ❌ Non-null assertions (`!`)
- ❌ Hardcoded strings (use i18n)
- ❌ Direct Supabase calls in components (use hooks)
- ❌ Global state (use React Query or local state)
- ❌ Magic CSS (use Tailwind utilities)
- ❌ Component prop drilling > 2 levels

**Encouraged patterns**:
- ✅ Zod validation on all external data
- ✅ Explicit error handling
- ✅ Clear loading/error/empty states
- ✅ WCAG ARIA patterns
- ✅ Feature-based folder organization
- ✅ Smart + presentational split
- ✅ Test-friendly hook design

## 10. Example: Complete Feature

**Feature: Enrolment (multi-step wizard)**

```
components/Enrolment/
├── EnrolmentStepper.tsx    # Smart: Manages wizard flow
├── StepClassSelect.tsx     # Smart: Filter + select classes
├── StepParentInfo.tsx      # Smart: Collect parent data
├── StepReview.tsx          # Presentational: Review enrolment
├── StepSuccess.tsx         # Presentational: Success message
└── index.ts

hooks/
├── useEnrolment.ts         # Wizard state management
├── useAvailableClasses.ts  # Filter by requirements
└── useEnrolmentSubmit.ts   # API submission

schemas/
└── enrolment.ts            # Validation schemas

pages/
└── EnrolmentPage.tsx       # PublicLayout + EnrolmentStepper
```

**In EnrolmentStepper.tsx** (smart component):
```tsx
export function EnrolmentStepper() {
  const { step, data, advance, submit } = useEnrolment();
  const { classes } = useAvailableClasses(data.selectedClassIds);
  
  return (
    <div>
      {step === 'class-select' && (
        <StepClassSelect 
          classes={classes} 
          onSelect={advance}
        />
      )}
      {step === 'parent-info' && (
        <StepParentInfo 
          onSubmit={advance}
        />
      )}
      {step === 'review' && (
        <StepReview 
          enrolmentData={data}
          onSubmit={submit}
        />
      )}
    </div>
  );
}
```

**In StepClassSelect.tsx** (smart component):
```tsx
export function StepClassSelect({ classes, onSelect }) {
  const [selected, setSelected] = useState<string | null>(null);
  const { t } = useTranslation();
  
  return (
    <div>
      <h2>{t('enrolment.select_class')}</h2>
      {classes.map(cls => (
        <ClassSelectCard 
          key={cls.id}
          class={cls}
          selected={selected === cls.id}
          onSelect={() => setSelected(cls.id)}
        />
      ))}
      <button 
        onClick={() => onSelect(selected)}
        disabled={!selected}
      >
        {t('common.next')}
      </button>
    </div>
  );
}
```

**In ClassSelectCard.tsx** (presentational):
```tsx
export function ClassSelectCard({ class: cls, selected, onSelect }) {
  return (
    <article 
      onClick={onSelect}
      role="button"
      className={selected ? 'border-primary' : 'border-gray-300'}
    >
      <h3>{cls.name}</h3>
      <p>{cls.start_time} – {cls.end_time}</p>
      <p>{formatCurrency(cls.price_minor)}</p>
    </article>
  );
}
```

## Quick Checklist

Before committing a new component:

- [ ] Component in feature folder (`components/[Feature]/`)
- [ ] Smart/presentational split clear
- [ ] Hook for business logic (`hooks/use[Feature].ts`)
- [ ] Schemas for validation (`schemas/[feature].ts`)
- [ ] Exported from `index.ts`
- [ ] WCAG ARIA patterns present
- [ ] Loading/error/empty states handled
- [ ] No hardcoded strings (all i18n)
- [ ] TypeScript strict mode (no `any`)
- [ ] Lines of code within limits
- [ ] Props well-typed with interfaces
- [ ] JSDoc comments on complex logic
