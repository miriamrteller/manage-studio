## 2. RTL & Israeli Localization (Strict)

### 2.1 Language Architecture (RTL/LTR)

**Single Source of Truth:** HTML `<html>` element only
- `<html lang="en" dir="ltr">` — default safe value at startup
- Updated dynamically by `useLanguage()` hook only
- Direction computed from language only: `language === 'he' ? 'rtl' : 'ltr'`
- Never pass `dir` or `country` to components

**Language State Hierarchy** (Precedence)
1. User language preference (if set in future)
2. Tenant `language_default` (from config)
3. App default: `'en'` (LTR)

**Usage:**
- `useLanguage()` hook — call once in App root component
- Call: `useLanguage()` at the start of `App.tsx`
- Hook watches `tenant.language_default` and auto-updates HTML dir + lang
- Components never set direction; they inherit from `<html>` via CSS cascade

**Frontend Never Computes Direction:**
- No `tenant.dir` property exists in TenantConfig
- No `country` property in TenantConfig
- No inline `dir={...}` attributes on elements
- No other direction-setting code except `useLanguage()` hook

### 2.2 Logical CSS (Required)
- FORBIDDEN: Physical CSS (`ml-*`, `pr-*`, `left`, `right`).
- MANDATORY: Logical CSS (`ms-*`, `pe-*`, `inset-inline-start`).
- Use `<bdi>` or `<bdo>` tags and Template Literals for mixed Hebrew/English strings.
- ALWAYS specify `Asia/Jerusalem` timezone using `date-fns`.

## 3. Law & Compliance (Israel/UK)
- Refunds: Logic MUST verify both the standard 14-day window AND the `is_protected_consumer` 4-month window.
- Invoices: Logic MUST verify the 20,000 NIS threshold to fetch ITA Allocation Numbers.

## 4. Design System & Colors (White-Label Compliance)
- **FORBIDDEN:** Hardcoded hex colors (`#76335a`), arbitrary Tailwind values (`text-[#...]`), inline color styles in JSX.
- **MANDATORY:** All colors via CSS variables (`var(--color-primary)`), semantic class names (`text-primary`, `bg-error`), validated with `TenantWhiteLabelSchema`.
- Color inputs validated in Zod schema before storage. All derived colors computed by `deriveColorSystem()` in `utils.ts`.
- **Build gate:** Changes to `src/lib/utils.ts` or color-related CSS must pass `pnpm run build` immediately.
- Multi-tenant rule: Never query tenant colors directly. Always load via `useTenant()` → `useThemeInjection()` pipeline.
</technical_standards>

## Accessibility Enforcement: WCAG 2.1 Level AA

**Mandate:** Accessibility is non-negotiable for Israeli community centers (legal requirement per דינ נגישות לאנשים עם מוגבלות, 1998). All UI features must pass WCAG 2.1 Level AA automated tests before merge, with additional manual verification pre-ship.

**3-layer enforcement:**
1. **Write-time (IDE feedback):** ESLint jsx-a11y warnings — non-blocking, helps developers catch patterns early.
2. **Merge-time (hard gate):** axe-core scans run in CI; zero violations required. Blocks PRs.
3. **Ship-time (manual):** 15-minute NVDA Hebrew smoke test + PM/QA sign-off. Ensures no algorithmic issues mask real UX problems.

### Setup (run once per workspace)

```bash
# Install accessibility tools
pnpm add -D -w @axe-core/react axe-playwright eslint-plugin-jsx-a11y

# Verify snyk compliance
pnpm dlx snyk test @axe-core/react axe-playwright eslint-plugin-jsx-a11y
```

### Manual smoke test (15 minutes — pre-deployment only)

**Tools:** NVDA (Screen reader) + keyboard-only navigation

**Test flow:**
1. **Heading structure (3 min):** Open DevTools → Elements, verify no h2/h3 without h1, no level skips
2. **Form errors (3 min):** Submit blank form, verify error messages read aloud + highlight visible
3. **Focus traps (2 min):** Open modal, Tab through all elements, Escape closes modal
4. **Contrast (3 min):** Use APCA contrast checker for all text (especially over images); ≥ 4.5:1 for body text
5. **Keyboard nav (2 min):** Tab through full page, no focus loss, all interactive elements reachable
6. **RTL Tab order (2 min):** Verify Tab moves right-to-left in Hebrew mode

**Sign-off:** Product Manager + QA confirm before merge to production.
