<technical_standards>
## 1. React & TypeScript
- NO `any`. NO non-null assertions (`!`). USE `unknown` or `never`.
- Force exhaustive switch statements for union types (include `default: const _exhaustiveCheck: never = ...`).
- File limit: 250 lines. Component limit: 150 lines.

## 2. RTL & Israeli Localization (Strict)
- FORBIDDEN: Physical CSS (`ml-*`, `pr-*`, `left`, `right`).
- MANDATORY: Logical CSS (`ms-*`, `pe-*`, `inset-inline-start`).
- Use `<bdi>` or `<bdo>` tags and Template Literals for mixed Hebrew/English strings.
- ALWAYS specify `Asia/Jerusalem` timezone using `date-fns`.

## 3. Law & Compliance (Israel/UK)
- Refunds: Logic MUST verify both the standard 14-day window AND the `is_protected_consumer` 4-month window.
- Invoices: Logic MUST verify the 20,000 NIS threshold to fetch ITA Allocation Numbers.
</technical_standards>
